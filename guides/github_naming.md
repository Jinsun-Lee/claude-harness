# 레포 명명 규칙
1. 기본적으로 `kebab-case`를 사용한다.
2. 일반 레포는 `{접두어}-{주제}[-{추가요소}]` 형식으로 작성한다.
3. `접두어`는 레포를 묶는 기준이며, 카테고리, 플랫폼, 제품명, 도메인이 될 수 있다.
4. `주제`는 레포가 무엇에 대한 것인지 나타내는 핵심 도메인이다.
5. `추가요소`는 타입, 버전, 공개 범위, 대상, 환경 등 같은 주제를 더 구체화할 때 사용한다.

<br>

## 접두어
1. 카테고리 접두어는 `study`, `lecture`, `project`, `job`, `race`를 사용한다.
2. `study`는 계속 관리하거나 참고하는 학습 자료에 사용한다.
3. `lecture`는 정해진 기간 내에 수강하는 과목에 사용한다.
4. `project`는 정해진 기간 내에 진행한 프로젝트에 사용한다.
5. `job`은 취업, 지원, 면접 준비 자료에 사용한다.
6. `race`는 자율주행 레이스, 경진대회, 경기 운영/참가 프로젝트에 사용한다.
7. 플랫폼 접두어는 `claude`, `github`, `youtube`를 사용한다.

<br>

## 템플릿
1. 템플릿 레포는 `{도메인}-template` 형식으로 작성한다.
2. `template`은 항상 suffix로만 둔다.
3. 현재 사용하는 템플릿은 `ai-template`, `cs-template`, `devops-template`, `embedded-template`, `misc-template`이다.

<br>

## ROS2
1. ROS2 레포는 빌드/워크스페이스 호환성을 위해 일반 규칙과 분리한다.
2. `ros2_{주제}_{추가요소}` 형식으로 작성한다.
3. `ros2_` 접두어를 유지한다.
4. `ros2_` 뒤쪽은 `snake_case`로 작성한다.

<br>

## 예외
1. GitHub 시스템, 개발환경 관례, 또는 짧은 고유명사 성격이 있는 레포는 이름을 유지한다.
2. 유지 대상은 `.github`, `Jinsun-Lee`, `Jinsun-Lee.github.io`, `dotfiles`, `gitbook`, `pin`, `tistory`이다.
