# Git
1. 파일 이동은 모두 `git mv`로 처리한다.
2. 과거 커밋을 수정할 때 원래 커밋 날짜와 시간을 유지한다.
   1. cherry-pick이나 rebase 시 `--committer-date-is-author-date` 옵션을 사용한다.
3. 다른 성격(예: fix, docs, feat)의 변경은 하나로 묶지 않고 각각 커밋한다.
4. 커밋 메시지는 `[prefix] 내용` 형식으로 작성한다.
   1. 제목은 변경 사항을 상세하게 작성한다.
   2. 내용이 많으면 본문에 `-` 불릿으로 추가한다.
   3. 추상적이거나 동일한 메시지를 반복하지 않는다.
5. 커밋을 만들면 별도 확인 없이 바로 push까지 이어서 실행한다.
