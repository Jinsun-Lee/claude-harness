# 코드 작성
## 작업 방식
1. **코드 수정 시 다음 항목을 상세히 여러 번 검토하여 파악한다.**
   1. 잠재 오류
   2. 신규 버그
   3. 사이드 이펙트
2. **결정·수정 안은 서브 에이전트 여러 개에게 검증받아 진행한다.**
3. **코드를 수정하기 전에 현재 파일/함수의 동작을 먼저 설명하고 변경 방향을 합의한다.**
4. **성능 최적화는 의도적 비효율일 수 있으므로 수정 전 합의받는다.**
5. **겹치지 않는 파일은 병렬로 동시에 수정한다.**
6. **파일 수정 후 작업 단위로 나눠 자주 커밋한다.**
7. 요청 범위만 수행한다.
8. 요청 외 파일·구현을 추가하지 않는다.

## 코드 품질
1. 과도한 추상화를 하지 않는다. 가장 단순한 작동 코드만 작성한다.
2. **재사용성이 있는 로직은 모듈화한 함수로 분리해 import해서 사용한다.**
3. 매직 넘버·고정값을 하드코딩하지 않는다.
4. **한 파일에 너무 많은 내용을 두지 않는다.**
5. **데드 코드를 제거하고 주석을 함께 갱신한다.**
6. **명명·표기·구조를 일관되게 유지한다.**

## 명명
1. 파일명은 snake_case로 통일한다.
2. 함수·변수는 snake_case로 작성한다. (Python PEP 8, C Linux 커널, C++ STL 풍)
3. 클래스·구조체는 PascalCase로 작성한다. (단 C 구조체는 snake_case)
4. 상수·매크로는 UPPER_SNAKE_CASE로 작성한다.
