# 코드 작성
## 작업 방식
1. 코드를 수정하기 전에 현재 파일/함수의 동작을 먼저 설명하고 변경 방향을 합의한다.
2. 요청 외 제안/리팩토링/최적화를 하지 않는다. 요청 범위만 수행한다.
3. 요청 외 파일을 수정하지 않는다.

## 코드 품질
1. 과도한 추상화를 하지 않는다. 가장 단순한 작동 코드만 작성한다.
2. 코드는 최대한 모듈화한다.
3. 코드는 모듈화한 함수를 import해서 사용한다.
4. 파일 마지막 줄에 빈 줄을 두지 않는다.

## 명명
1. 파일명은 snake_case로 통일한다.
2. 식별자는 snake_case, 클래스는 PascalCase, 상수·매크로는 UPPER_SNAKE_CASE로 작성한다. (Python/C/C++ 공통)
